# 网络安全攻击知识库系统需求设计文档

## 1. 项目概述

### 1.1 项目名称
网络安全攻击知识库系统（Attack Knowledge Graph System）

### 1.2 项目背景
面对政务信息化领域的网络安全威胁识别和溯源分析需求，基于MITRE ATT&CK框架构建一套面向真实世界观察的对手战术和技术的全球可访问知识库体系，为政务信息化搭建体系化防御体系提供支撑。

### 1.3 项目目标
- 构建基于ATT&CK框架的网络安全知识库体系
- 提供100+威胁行为者组织的详细信息
- 支持威胁识别、防御漏洞评估、安全工具能力评估
- 支持检测组织、威胁寻找、红队活动、缓解控制验证
- 提供交互式ATT&CK图谱可视化

### 1.4 技术栈
- **前端**: Vue 3 + TypeScript + Element Plus + ECharts/D3.js
- **后端**: Flask + Python 3.9+
- **数据库**: PostgreSQL + Redis
- **ATT&CK集成**: mitreattack-python库
- **可视化**: D3.js / Cytoscape.js / ECharts

## 4. 数据模型设计

### 4.1 核心实体

#### 4.1.1 战术（Tactic）
```
- id: 唯一标识
- tactic_id: MITRE战术ID (如: TA0001)
- name: 战术名称
- description: 战术描述
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.2 技术（Technique）
```
- id: 唯一标识
- technique_id: MITRE技术ID (如: T1001)
- name: 技术名称
- description: 技术描述
- tactic_id: 所属战术ID
- platforms: 适用平台
- permissions_required: 所需权限
- data_sources: 数据源
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.3 子技术（SubTechnique）
```
- id: 唯一标识
- subtechnique_id: MITRE子技术ID (如: T1001.001)
- name: 子技术名称
- description: 子技术描述
- technique_id: 所属技术ID
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.4 威胁行为者（ThreatActor）
```
- id: 唯一标识
- actor_id: MITRE行为者ID
- name: 行为者名称
- aliases: 别名列表
- description: 行为者描述
- first_seen: 首次发现时间
- last_seen: 最近活动时间
- motivation: 动机
- origin: 来源地
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.5 软件（Software）
```
- id: 唯一标识
- software_id: MITRE软件ID
- name: 软件名称
- type: 类型 (malware/tool)
- description: 软件描述
- platforms: 适用平台
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.6 缓解措施（Mitigation）
```
- id: 唯一标识
- mitigation_id: MITRE缓解措施ID
- name: 缓解措施名称
- description: 缓解措施描述
- created_at: 创建时间
- updated_at: 更新时间
```

#### 4.1.7 检测规则（DetectionRule）
```
- id: 唯一标识
- name: 规则名称
- description: 规则描述
- rule_type: 规则类型 (signature/behavior)
- content: 规则内容
- technique_ids: 关联技术ID列表
- severity: 严重级别
- status: 状态
- created_by: 创建者
- created_at: 创建时间
- updated_at: 更新时间
```

## 5. 接口设计

### 5.1 RESTful API规范

#### 5.1.1 ATT&CK数据接口
```
GET /api/v1/tactics - 获取战术列表
GET /api/v1/tactics/{id} - 获取战术详情
GET /api/v1/techniques - 获取技术列表
GET /api/v1/techniques/{id} - 获取技术详情
GET /api/v1/subtechniques - 获取子技术列表
GET /api/v1/subtechniques/{id} - 获取子技术详情
```

#### 5.1.2 威胁行为者接口
```
GET /api/v1/actors - 获取威胁行为者列表
GET /api/v1/actors/{id} - 获取威胁行为者详情
GET /api/v1/actors/{id}/techniques - 获取行为者使用的技术
GET /api/v1/actors/{id}/software - 获取行为者使用的软件
```

#### 5.1.3 检测规则接口
```
GET /api/v1/rules - 获取规则列表
POST /api/v1/rules - 创建规则
PUT /api/v1/rules/{id} - 更新规则
DELETE /api/v1/rules/{id} - 删除规则
POST /api/v1/rules/{id}/test - 测试规则
```

#### 5.1.4 报告接口
```
GET /api/v1/reports - 获取报告列表
POST /api/v1/reports - 生成报告
GET /api/v1/reports/{id} - 获取报告详情
GET /api/v1/reports/{id}/download - 下载报告
```

#### 5.1.5 可视化接口
```
GET /api/v1/visualization/matrix - 获取ATT&CK矩阵数据
GET /api/v1/visualization/attack-path - 获取攻击路径数据
GET /api/v1/visualization/relationships - 获取关联关系数据
```

#### 5.1.6 同步接口
```
POST /api/v1/sync/attack - 同步ATT&CK数据
GET /api/v1/sync/status - 获取同步状态
```

│  └──────────┴──────────┴──────────┴──────────┴──────────┘  │

                              │
